# The PASS / REVISE / REJECT Decision System

Every Taste Engine audit resolves to exactly one decision. The decision is the product — the score is just how we got there.

---

## The three decisions

### PASS

The asset is strong enough to ship.

It clears the brand, design, audience, and production bar without changes that would meaningfully improve outcomes. Minor polish may still be suggested, but none of it blocks shipping.

> Default numeric band: **85–100**.

### REVISE

The concept is viable, but execution needs targeted changes.

The idea is worth keeping; specific, nameable problems stand between it and shipping. A REVISE always comes with prioritized fixes and, where useful, an improvement brief or regeneration prompt. REVISE is not a soft REJECT — it's a vote of confidence in the concept.

> Default numeric band: **60–84**.

### REJECT

The concept or execution fails core standards.

Either the idea is wrong for the brand/audience/medium, or the execution is broken in ways that aren't worth salvaging. REJECT should explain what would need to be true for the concept to be worth revisiting — or state plainly that it shouldn't be.

> Default numeric band: **0–59**.

---

## Override rules

Score sets the default decision. A fatal issue overrides it. When an override fires, the decision becomes **REJECT** regardless of the numeric score, and the audit must state which override applied and why.

- **Fatal brand mismatch** — the asset contradicts core brand positioning, audience, or "never do this" constraints. A polished asset for the wrong brand still fails.
- **Fatal production issue** — the asset cannot be produced or implemented in its intended medium (unprintable detail, color count over the process limit, breaks at the required size, illegible at distance/thumbnail).
- **Legal / IP concern** — the asset risks trademark, copyright, likeness, or regulatory violation. This is a hard stop, not a design note.
- **Severe clarity failure** — a first-time viewer cannot tell what the asset is or what to do, within the medium's attention window. No amount of polish overrides broken communication.

Overrides only ever move a decision *down* (toward REJECT). A fatal issue can never be outweighed by a high score in other categories — that's the entire point of an override.

---

## Why a gate, not a grade

A grade describes an asset. A gate decides its fate. Taste Engine is a gate: in a pipeline producing hundreds of assets, the value is a consistent, explainable PASS / REVISE / REJECT applied every time — so weak work stops before it ships, and strong work moves without a human reviewing each one by hand.
